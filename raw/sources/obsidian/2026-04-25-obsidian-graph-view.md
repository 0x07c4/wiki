# Obsidian Graph View Source Note

Captured: 2026-04-25  
Purpose: confirm Obsidian's native graph capability for the local agent knowledge base workflow.

This is a curated source note, not a full mirror of external pages. External pages remain the source of record.

## Sources

- Obsidian Help, "Graph view".
  URL: https://help.obsidian.md/plugins/graph
  Captured facts:
  - Graph view is a core plugin.
  - It visualizes relationships between notes in a vault.
  - Notes are represented as nodes, and internal links are represented as lines.
  - Graph settings include filters, groups, display options, and force controls.
  - Local Graph shows notes connected to the active note and supports configurable depth.

- Obsidian Help, "How Obsidian stores data".
  URL: https://help.obsidian.md/data-storage
  Captured facts:
  - Obsidian stores notes as Markdown files in a local vault.
  - Obsidian maintains a metadata cache that powers features including Graph view and Outline view.

## Initial Reading

- Obsidian's graph capability is native to the app's core-plugin layer, not merely a third-party plugin add-on.
- For this repository, Obsidian should be described as the human-facing workspace with built-in graph navigation.
- Additional plugins should be framed as workflow extensions around source intake, dashboards, review, presentation, or agent handoff.
