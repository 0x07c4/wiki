# Source: Obsidian Agent Workbench Skeleton 2026-04-26

## Summary

This local source note captures the first read-only Obsidian Agent Workbench plugin skeleton. It matters because the wiki now has a concrete human-facing workbench implementation path that consumes `llm-wiki` command handoffs without writing to `raw/`, `wiki/`, or `.obsidian/`.

## Key Claims

- The plugin skeleton lives under `tools/obsidian-agent-workbench/`.
- It registers an `Agent Workbench` side view and command-palette entry points.
- It reads active markdown page metadata and outbound links from Obsidian.
- It prepares read-only `llm-wiki` JSON command handoffs.
- It does not write wiki content or raw sources.
- TypeScript typecheck and esbuild build passed.

## Evidence Notes

- `manifest.json` declares a desktop-only Obsidian plugin.
- `src/main.ts` implements the side view, settings, and command copying.
- `package.json` defines `typecheck` and `build` scripts.
- `main.js` is generated by build and ignored by git.
