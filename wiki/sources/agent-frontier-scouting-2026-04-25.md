# Source: AI Agent Frontier Scouting Pack 2026-04-25

## Summary

This source pack collects recent AI agent developments across frontier models, coding agents, workspace agents, SDKs, skills, standards, and evaluation. The main signal is that the field is moving from chat-plus-tools toward runtime-backed systems with sandboxed execution, shared agents, memory, skills, approvals, and agent-specific evaluation.

## Key Claims

- frontier model releases increasingly emphasize agentic coding, computer use, long-horizon execution, and knowledge-work workflows
- OpenAI, Anthropic, and Google are all packaging model capability inside agent runtime or product infrastructure rather than presenting models alone
- skills are becoming a portable way to package procedural knowledge, scripts, templates, and domain-specific workflows for agents
- open standards such as MCP, AGENTS.md, and Agent Skills are becoming important coordination layers between tools, models, and workspaces
- evaluations are shifting toward realistic execution: terminal tasks, software issue resolution, GUI control, tool-agent-user interaction, reliability, and latency

## Evidence Notes

- OpenAI's 2026 releases connect Codex, workspace agents, Agents SDK, sandbox execution, skills, memory, automations, and enterprise governance
- Anthropic's 2025 Claude 4.5 and Skills releases emphasize long-running code work, computer use, progressive disclosure, and filesystem-based skill packaging
- Google releases position Gemini models around computer use, deep research agents, agentic development, and UI/tool loops
- Terminal-Bench 2.0, SWE-Bench Pro, τ-bench, and OSWorld-Human each expose gaps that simple coding or answer benchmarks do not cover
